# 提示词工作台实施会话启动指令

把下面的“推荐启动指令”完整粘贴到新的 Codex 对话。新对话不需要拥有之前的聊天记录，仓库文档和参考图是事实来源。

## 推荐启动指令

```text
你现在要在 D:\test项目\sub2Image 中实现“通用 ConversationComposer + 提示词工作台 Tool”。

开始任何修改前，必须完整阅读并遵守：
1. D:\test项目\sub2Image\AGENTS.md
2. D:\test项目\sub2Image\docs\prompt-studio-design.md
3. D:\test项目\sub2Image\docs\prompt-studio-development-plan.md
4. D:\test项目\sub2Image\docs\assets\prompt-studio-composer-reference.png
5. D:\test项目\sub2Image\docs\assets\prompt-studio-parallel-composer-reference.png

这些文档是跨会话事实来源，不要依赖当前对话之外的记忆。用户最新消息优先；如果最新要求改变了已确认设计，先同步更新两份文档再实现。

执行规则：
- 先检查 git status，保留所有现有用户改动，不覆盖、不回滚无关文件。
- 查看开发计划第 1.3 节“执行状态”，从第一个未完成且依赖已满足的工作包开始。
- 开始工作包时把状态改成“进行中”。
- 一次只完成一个工作包，不跨阶段混改；严格遵守该工作包的目标、涉及文件、测试门禁和回滚点。
- 不进行一次性重写 InputBar，不向 store.ts 塞入 Prompt Studio 领域逻辑。
- 开发和验收期间保留底部原 InputBar，把新 ConversationComposer 挂载在它正上方；两者同时可用，全部门禁通过且用户明确验收后才允许删除旧输入框。
- 新旧输入框不得双重提交或同时响应同一次 paste/drop/快捷键；WP2 不显示空壳，WP3 具备最小可操作能力后再挂载验收区。
- ConversationComposer 不得依赖具体 Tool；Prompt Studio、图片生成和普通对话通过公开 Tool 协议接入。
- 提示词 Tool 默认关闭，只有用户主动开启后才运行。
- 模型只优化和补充建议，最终提示词由用户决定。
- 应用提示词后只写回 Composer 并切换到图片 Tool，不自动提交；用户再次发送才生成。
- 视频、历史图片任务入口和全局项目列表不属于当前范围，不要顺手实现。
- 遇到不确定行为先查现有源码和测试；没有证据时输出必要日志让我提供，不要增加猜测式兼容代码。
- Windows 常驻服务必须后台启动，stdout/stderr 写入日志，并报告 PID、日志路径和端口检查结果。
- 前端交互修改必须使用 Playwright 验证桌面和移动端；引用图片必须实际检查是否显示。
- 每个工作包完成前运行该节指定测试，并至少运行 npm run build 和 npm test。
- 只有全部门禁通过后，才把状态改为“已完成”，同时写入验证命令、通过数量和必要截图路径。
- 不要创建 git commit、分支或推送，除非我明确要求。

现在先汇报你识别到的当前工作包、依赖和预计修改文件，然后直接开始实现，不要只给方案。完成该工作包后停止并给出：改动摘要、测试结果、风险和下一个工作包。
```

## 继续下一个工作包

当前工作包完成后，在同一对话或新的对话中使用：

```text
继续按照 D:\test项目\sub2Image\docs\prompt-studio-development-plan.md 执行。先读取第 1.3 节状态和最近 Git 差异，从第一个未完成且依赖已满足的工作包开始。仍然一次只完成一个工作包，全部门禁通过后更新状态和验证证据。
```

## 连续执行模式

不推荐在第一个实施会话就连续执行全部工作包。如果已经完成 WP0～WP5、回归稳定，并希望一个会话继续多个工作包，可以在推荐启动指令最后追加：

```text
当前工作包全部门禁通过并更新状态后，可以继续下一个依赖已满足的工作包；每个工作包仍需独立验证和记录，不得合并跳过。遇到需要产品决定、数据迁移异常或无法解释的回归时停止并向我汇报。
```
