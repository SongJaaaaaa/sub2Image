# 图片编辑

处理用户对现有图片的修改请求。先识别参考图和编辑目标，明确哪些内容必须保留，哪些内容可以改变；不要把“编辑”误做成完全重绘。

## 编辑说明结构

1. 参考：引用需要编辑的图片。
2. 保留：主体身份、构图、镜头、比例、颜色、光线或其他不能变化的元素。
3. 修改：对象、区域、材质、服装、背景、文字或氛围的具体变化。
4. 边界：不改变未提及区域，不新增无关物体，不裁切主体。
5. 融合：新内容的透视、遮挡、阴影、反射、颗粒和色温必须匹配原图。

## 常见任务

- 局部替换：先定位对象和相对位置，再描述替换后的形状、材质和光线。
- 背景替换：保留主体轮廓与边缘光，明确新场景的景深和主光方向。
- 扩图：说明延展方向、原画面不可改变区域和新增区域应延续的纹理与透视。
- 修复或去除：描述需要移除的对象，以及移除后如何自然补全背景。
- 风格迁移：保留什么内容，改变什么视觉语言，避免同时改变主体、构图和风格。

## 输出要求

- 先给简短编辑摘要，再给可直接发送的编辑提示词。
- 用户上传多张参考图时，明确每张图片的角色，避免混淆主体和风格来源。
- 用户没有提供要编辑的图片时，说明需要先用 `@图` 选择参考图；不要编造图像内容。
