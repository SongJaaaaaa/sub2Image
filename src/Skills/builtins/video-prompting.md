# 视频生成提示词

将用户的短视频想法转成模型可执行的镜头提示词。当前应用没有视频生成工具时，只输出视频提示词、关键帧和分镜，不得声称已经生成视频；用户同时需要封面或关键帧时，才使用图像生成能力。

## 先确定

- 成片用途、比例、时长、帧率感受和风格。
- 主体、场景、起始状态、结束状态和唯一核心动作。
- 镜头语言：固定、推拉、平移、跟拍、环绕、手持或俯拍。
- 节奏：0 到 2 秒建立画面，中段发生什么，结尾停在什么状态。
- 需要保持一致的角色、服装、道具、光线和场景锚点。

## 视频提示词结构

按“主体与场景 / 起始画面 / 动作过程 / 镜头运动 / 光线与风格 / 结束画面 / 时序限制”输出。

示例结构：

```text
[主体与场景]。开始时 [起始状态]；随后 [连续动作]。镜头 [单一镜头运动]，保持 [一致性锚点]。光线与风格为 [视觉语言]。结尾停在 [结束状态]。避免 [闪烁、变形、人物身份漂移、镜头突变等具体问题]。
```

## 约束

- 一条短视频优先一个核心动作和一个镜头运动，避免“同时变形、转场、换景、换装、多人互动”。
- 动作按时间顺序描述，使用连续动词，不堆砌静态形容词。
- 图生视频必须说明哪张图是首帧或角色参考，以及哪些元素不能变化。
- 多镜头视频先输出分镜表；每个镜头单独写提示词和衔接状态。
- 不虚构模型不支持的音频、口型、物理或时长控制能力。
