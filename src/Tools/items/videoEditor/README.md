# 视频剪辑

视频剪辑 Tool 使用 `@xzdarcy/react-timeline-editor` 提供时间轴交互，使用 `ffmpeg.wasm` 完成浏览器端的裁剪、拼接、音频处理和导出。所有素材只在当前浏览器中处理。

## 当前实现

- 支持导入 MP4、WebM、MOV，读取时长、尺寸和常见视频编码。
- 单轨片段可分割、两端裁剪、删除、拖动排序和按钮排序。
- 支持添加多个图片遮盖层，每层可在轨道上移动、裁剪播放区间，并调整位置、大小、旋转、透明度和上下层级。
- 支持导入 UTF-8 SRT/VTT 字幕，在字幕轨道调整时间和文字，并设置字号、颜色、背景和位置。
- 字幕在预览中按时间显示，导出时通过透明文字图层烧录进 MP4，中文字体由浏览器渲染。
- 导入时自动抽取代表帧，时间轴以胶片形式展示，裁剪和分割后同步更新可见帧。
- 视频导入时显示解析与抽帧进度，不再以无反馈的等待状态阻塞操作。
- 视频预览、进度条和时间轴游标同步；每个片段使用独立播放器并预热下一片段，跨剪切点时确认首帧后切换并保持上一帧。
- 支持原声音量与静音；背景音乐拥有独立轨道，可移动、裁剪播放区间，超出音乐源长度时自动循环拼接。
- 支持 16:9、9:16、1:1 画布和 720p、1080p 导出分辨率。
- 导出时懒加载本地 `ffmpeg.wasm`，生成 H.264/AAC MP4 并显示进度。
- 可将剪辑结果连同封面保存到 IndexedDB，并作为视频任务在画廊中播放或下载。

依赖许可证：`@xzdarcy/react-timeline-editor`、`@ffmpeg/ffmpeg`、`@ffmpeg/util` 为 MIT；实际编解码核心 `@ffmpeg/core` 为 GPL-2.0-or-later。

## 第一版：基础剪辑（已实现）

- 上传 MP4、WebM、MOV 等常见视频，并读取时长、尺寸和编码信息。
- 视频预览与时间轴同步，支持播放、暂停和拖动定位。
- 单轨视频的分割、裁剪、删除、排序和多片段拼接。
- 原声音量调整、静音、添加一条背景音乐和音频裁剪。
- 选择横屏、竖屏、方形比例，设置基础导出分辨率。
- 使用 `ffmpeg.wasm` 导出 MP4，显示明确的处理进度和失败信息。
- 小视频在浏览器本地处理，不上传到业务服务器。

第一版暂不包含转场、滤镜、多画面叠加和视频抠图。

## 第二版：内容增强

- 多轨视频、图片和音频素材编排。
- 文字标题轨道和更丰富的字幕样式。
- 淡入淡出、叠化等常用转场。
- 画面裁剪、旋转、速度、音量关键帧和常用滤镜。
- 音频波形、淡入淡出、背景音乐循环和多音轨混合。
- 封面选择、草稿保存、撤销和重做。

## 第三版：大文件与智能能力

- 大文件和长视频改用服务端 FFmpeg 队列导出，浏览器保留编辑与预览。
- 导出任务断点恢复、进度查询、取消和结果保存。
- 自动字幕、静音检测、场景切分等辅助能力。
- 视频抠图使用独立的时序 Matting 模型，不通过图片抠图逐帧处理。

## 技术边界

```text
时间轴状态
  -> 浏览器 video / Canvas 预览
  -> FFmpeg 参数生成
  -> ffmpeg.wasm 本地导出
  -> 服务端 FFmpeg 大文件导出（第三版）
```

- `react-timeline-editor` 只负责时间轴交互，不负责媒体处理。
- `ffmpeg.wasm` 负责实际裁剪、拼接、混音和转码，但大视频受浏览器内存限制。
- 时间轴数据与 FFmpeg 参数转换应放在本 Tool 内，访问共享资源时只能通过 `Tools/adapters`。
- 引入依赖前需要再次确认许可证以及 FFmpeg 构建中启用的编解码器授权。
